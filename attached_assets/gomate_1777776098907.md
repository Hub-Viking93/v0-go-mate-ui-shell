# GoMate v2 — Replit Agent Prompt Playbook

This file is prompts. Paste each block into Replit Agent in order. Don't paraphrase. Don't shorten. Replit Agent needs the verbose context.

There are 41 prompts grouped in 9 phases. Each prompt is focused on a single deliverable. The Spine + Execution Waves section below tells you which 24 are the minimum viable path and which 17 are optional polish. Pre-flight checklist + story arc reference live at the end. Story arc is for you, not Replit.

After each prompt completes, BEFORE pasting the next one, run a quick verification: ask Replit Agent to confirm what it built and what's still pending. If it says "I'll come back to that later" — push back, force completion before moving on.

The lifecycle these prompts deliver:
- Stage 1 collecting → user fills profile via mascot onboarding (multi-agent extraction)
- Profile complete → user clicks "Generate my plan" → Stage 2 generating runs (multi-agent research with visible panels)
- Research complete → Stage 3 ready_for_pre_departure → user clicks "Generate my pre-departure checklist" → Stage 4 pre_departure (timeline of actions weeks-before-move)
- User does pre-departure work → clicks "I have arrived" when they land → Stage 5 arrived (settling-in DAG generated, post-arrival chat with smart suggestions)

Two manual generation triggers: research generation, pre-departure generation. This gives the user control + creates natural waypoints where users see each phase fire.

GoMate is not a "demo" or a "proof of concept" — it is an operating product with real users on a waitlist. This buildathon is fixing existing bugs and shifting positioning from "Relocation Assistant" (we help you find info) to "AI Relocation Consultant" (we replace the $5–15k human consultant). Do not call any part of this work a demo in commits, copy, or comments.

# THE SPINE — locked user journey (sits on the wall)

Every prompt is judged against this question: "Does this make the Spine stronger, or distract from it?" If it distracts, cut it.

The Spine is the 10-step user journey that the buildathon submission video walks through and the judges experience:

1. Land on `/` — see the new positioning copy
2. Click "Start your relocation plan"
3. Enter bundled answer in the mascot UI ("My name is Axel, I'm 30, moving from Germany to Sweden for a 14-month work posting with my wife and two kids")
4. Watch real-time extraction populate the dashboard with audit icons appearing as fields fill
5. Confirm or fill one missing field via the Question Director
6. Click "Generate my plan"
7. Agent panels populate live — six specialists working in parallel, dispatch logic visible (Visa Specialist, Tax Strategist, Cost-of-Living Analyst, Housing Researcher, Cultural Adapter, Documents Specialist)
8. Open the relocation plan with citations linking to Migrationsverket, Skatteverket, and other official sources
9. Click an audit icon → see the chain of evidence (which agent found this, when, from which URL)
10. Click "Generate my pre-departure checklist" → see consultant-grade timeline (apostille deadlines, A1 certificate, document templates, scheduled reminders) — the proof that GoMate replaces a consultant, not just a search tool

Total runtime: ~90 seconds. This is what the submission video records and what the judges walk through.

The marketing pitch — "Replace your $8,000+ relocation consultant for $39/month" — does NOT live inside the operating product. It belongs on gomaterelocate.com (homepage hero, pricing page, /why-gomate). The in-app `/compare` page (built by Prompt 9.1) is for architecture transparency only (single-agent bug vs multi-agent fix), not value-prop marketing.

# PHASE 5 — Pre-departure phase (NEW)

## Prompt 5.1 — Build the pre-departure timeline computation library

```
Build lib/gomate/pre-departure.ts.

This module is responsible for computing the pre-departure timeline based on user profile and visa pathway. It does NOT call agents directly — it's a pure function that gathers contributions from multiple specialist agents and orders them into a timeline.

Function signature:
```
async function generatePreDepartureTimeline(
  profile: Profile,
  selectedVisaPathway: VisaOption,
  moveDate: Date,
  specialistOutputs: { name: string, output: SpecialistOutput }[]
): Promise<PreDepartureTimeline>

type PreDepartureTimeline = {
  actions: PreDepartureAction[];
  totalActions: number;
  longestLeadTimeWeeks: number;
  criticalPath: PreDepartureAction[]; // actions on the critical path (delay these = delay arrival)
}

type PreDepartureAction = {
  id: string;
  title: string;
  description: string;
  weeksBeforeMoveStart: number;
  weeksBeforeMoveDeadline: number;
  estimatedDurationDays: number;
  dependsOn: string[]; // other action IDs
  documentsNeeded: string[];
  officialSourceUrl: string | null;
  preFilledFormUrl: string | null;
  agentWhoAddedIt: string;
  legalConsequenceIfMissed: string;
  status: "not_started";
  sortOrder: number;
}
```

Each specialist that has time-sensitive pre-departure actions contributes:

Visa Specialist contributes (if visa needed):
- Visa application starts (typically 8-12 weeks before)
- Required document gathering (varies by pathway)
- Visa application submission (4-6 weeks before)
- Embassy/consulate appointment if required
- Visa pickup before departure

Documents Specialist contributes:
- Birth certificate apostille (6-8 weeks lead time in some countries — applied for at start of timeline)
- Marriage certificate apostille (if applicable)
- Diploma apostille (if visa requires)
- Police clearance application (4-6 weeks)
- Translation chains for each apostilled document
- Pre-filled draft forms downloads

Schools Specialist contributes (if children):
- School applications begin (some international schools require 6-12 months lead time)
- Application document submission deadlines
- Acceptance confirmation milestones
- Final placement confirmation

Tax Strategist contributes (if departure tax applies):
- Departure declaration filing (typically end of tax year before move)
- Capital gains realization decisions
- Pension fund decisions

Banking Helper contributes:
- Bridge account setup (Wise/Revolut) — start now
- Notify origin bank of move
- Set up international transfer capability

Pet Specialist contributes (if pets):
- Microchip verification (if not already done)
- Rabies vaccine timing (21-day post-vaccination wait for EU travel)
- EU pet passport application (if EU)
- Pre-flight vet check
- Animal export permit if required

Posted Worker Specialist contributes (if posting):
- A1 certificate application (origin social security agency, 4-6 weeks)
- Posted Worker Declaration filing (must be before work starts, typically 2-4 weeks)
- Employer contact person designation
- Insurance arrangement for posting period

Healthcare Navigator contributes:
- Health insurance for travel period
- Medication supply (3-month supply if possible — some medications hard to get in destination)
- Medical record transfer requests

Vehicle Import Specialist contributes (if bringing vehicle):
- Customs declaration prep
- Emissions compliance check (technical inspection)
- Shipping or driving arrangement
- License plate cancellation in origin

Departure Tax Specialist contributes (if applicable):
- Final tax year filing
- Pension and retirement account decisions

Always-applicable timeline items (added by a generic "Departure Coordinator" sub-routine):
- 12 weeks before: Initial visa eligibility check (already done in research, but listed as confirmation)
- 8-10 weeks before: Major lead-time items (schools, apostille, A1)
- 4-6 weeks before: Visa submission, banking pre-setup, health insurance
- 2-4 weeks before: Address registration changes in origin (Abmeldung in Germany), mail forwarding, lease termination
- 1-2 weeks before: Final documents check, transit logistics, day-1-priorities printed
- Day of move: Travel checklist, what's in carry-on (passports, visa stickers, A1 certificates, all apostilled originals)

For each action:
- Compute weeksBeforeMoveStart from moveDate.
- Determine dependencies: e.g., visa submission depends on apostille completion + police clearance; A1 certificate depends on employer designation; school placement confirmation depends on application submission.
- Sort by weeksBeforeMoveStart descending (earliest first).
- Compute critical path using topological sort of dependencies + earliest start times.

Return the timeline. Don't write to database here — that happens in the next prompt.

Run pnpm tsc --noEmit.

When you're done, generate the pre-departure timeline for the German posting profile (move date 6 months from now) and the Filipino dependent profile (move date 4 months from now). Show me both timelines.

## Prompt 7.2 — Build the settling-in coordinator and DAG stitching

```
Build lib/agents/settling_in/coordinator.ts.

The Coordinator collects all specialist outputs and stitches them into a single DAG using existing dag-validator.ts.

Function:
```
async function stitchDAG(
  taskGroups: SettlingInTaskGroup[],
  profile: Profile,
  arrivalDate: Date
): Promise<{ tasks: SettlingTask[], dagValid: boolean, urgentDeadlines: SettlingTask[] }>
```

Logic:
1. Collect all tasks from all task groups.
2. Resolve cross-domain dependencies. Examples:
   - Banking tasks depend on Registration tasks (need personnummer).
   - Salary setup depends on Banking tasks.
   - School in-person registration depends on Address registration (need address proof).
   - Transit pass depends on Address registration (need to register at the resident pass).
   - BankID depends on Bank account + Government ID.
3. Topologically sort tasks using dag-validator.ts.
4. Compute deadlines (deadline_days after arrival_date).
5. Identify urgent deadlines: hard legal deadlines within 7-14 days (folkbokföring in Sweden is 7 days, Anmeldung in Germany is 14 days).
6. Mark legal requirements vs. recommended.
7. Return the sorted task list.

Replace lib/gomate/settling-in-generator.ts logic. The new generator function:
1. Calls all settling-in specialists in parallel.
2. Calls Coordinator to stitch DAG.
3. Saves all tasks to settling_in_tasks table (existing migration).
4. Returns the task list.

Update app/api/settling-in/generate/route.ts to use the new generator. The route's behavior:
1. Auth check.
2. Verify plan stage = "arrived" (this only runs when user clicks "I have arrived").
3. Call new generator.
4. Set settling_in_generation_completed = now() on relocation_plans.
5. Return tasks.

Update app/api/settling-in/arrive/route.ts:
