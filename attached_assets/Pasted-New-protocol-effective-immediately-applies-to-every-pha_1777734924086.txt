New protocol effective immediately, applies to every phase from this                                                  
  point forward (Phase 1 final completion through Phase 9):                                                             
  
  PHASE GATE TESTING — at the end of every phase, before moving to                                                      
  the next phase, you MUST write and run automated tests that prove
  the phase's deliverables work end-to-end. NOT manual walk-throughs.                                                   
  NOT typecheck-only verification. Real automated tests.                                                                
                                                                                                                        
  For each phase, produce two test artifacts:                                                                           
                                                                                                                        
  1. FRONTEND E2E (Playwright) — at artifacts/gomate/e2e/phase-N.spec.ts                                                
     - Boot the dev server fresh                                  
     - Sign in (or create test account via test fixture)                                                                
     - Walk through the user-visible flows the phase added or changed                                                   
     - Assert UI states (text content, element visibility, classnames)                                                  
     - Take screenshots at key moments (saved to /artifacts/screenshots/phase-N/)                                       
     - Click real buttons. Type real input. Wait for real network requests.                                             
     - At least 3 distinct test scenarios per phase (happy path + 2 edge cases)                                         
                                                                                                                        
  2. BACKEND INTEGRATION (vitest or node:test) — at                                                                     
     artifacts/api-server/test/phase-N.integration.test.ts                                                              
     - Hit real API routes with authenticated fetch                                                                     
     - Assert response shape, status codes, and timing                                                                  
     - Verify DB state changes (new rows, updated fields, RLS scoping)                                                  
     - Test failure paths (auth missing, validation fails, downstream                                                   
       dependency unavailable)                                                                                          
     - At least 2 distinct scenarios per significant API endpoint                                                       
                                                                                                                        
  Acceptance criteria for "phase complete":                                                                             
  - All Playwright tests pass (no flakes — re-run twice; both must pass)
  - All backend integration tests pass                                                                                  
  - pnpm tsc --noEmit passes                                                                                            
  - Screenshots saved are visually correct (you describe them; user                                                     
    spot-checks on request)                                                                                             
  - A PHASE-N-TESTS.md report is written summarizing: tests written,                                                    
    what each verifies, pass/fail status, any flake rates                                                               
                                                                                                                        
  Hard rules:                                                                                                           
  - Tests must use REAL data flows. No mocking the agent calls. If you                                                  
    need to keep test latency reasonable, use the cheapest real model                                                   
    (Haiku 4.5) and a small profile fixture — but the LLM round-trip is                                                 
    real.                                                                                                               
  - Tests must use REAL Supabase. Per-test cleanup in afterEach to                                                      
    remove fixtures.                                                                                                    
  - If a test fails, do NOT skip with .skip. Fix the bug. Tests are                                                     
    the truth, not a target to game.                                                                                    
  - If a test reveals a design flaw (not a bug — an actual "this can't                                                  
    be tested cleanly because the design is wrong"), pause and report                                                   
    back to user before proceeding.
